# Webクライアント UIガイドライン v0.1

- 更新日: 2026-02-26
- RUN_ID: `20260226T102426Z`
- ステータス: Draft（運用開始）
- 適用対象: Webクライアント全体（トップバー、ナビ、一覧、フォーム、モーダル、通知、グラフ）

## 目的と適用範囲

本ガイドラインは、業務用Web画面として「読みやすさ」と「操作の分かりやすさ」を優先しながら、見た目を安定させるために定める。

例外は、ブランド表現やキャンペーンなど、意図的に強い見た目が必要な限定ページに限る。

## 画面全体の設計方針

見た目は装飾量ではなく、情報の優先順位が自然に伝わるかで評価する。重要情報が先に見え、次に必要な操作へ視線が流れる状態を目標とする。

以下4点を固定方針とする。

- 重要操作と補助情報の見た目を同一にしない
- 色の薄さで迷わせない。境界と文字は明瞭にする
- 形と余白のルールを固定し、場当たり的な値を増やさない
- 状態（通常・選択・エラー等）を色だけで伝えない

## 数値で判断する品質基準

国際基準（WCAG）に沿い、以下を最低基準とする。

- 通常サイズ文字: 背景とのコントラスト比 `4.5:1` 以上
- 非テキスト識別要素（入力枠線、ボタン輪郭、選択状態など）: `3:1` 以上
- クリック/タップ対象: 原則 `24 x 24` CSS px 以上
- 必須・エラー・選択中など: 色だけに依存せず、形・アイコン・文言を併用

## 開発時の決め方（トークン運用）

色・余白・角丸などの値は、直接CSSへ書かずトークンで定義する。

- トークンは `:root` の CSS変数で管理する
- ページ固有の変数は原則作成しない
- 例外的に追加する場合は、理由と適用範囲をコメントで残す

## 基礎ルール

### 色

役割を固定し、色数を増やしすぎない。

- 土台（ページ背景）
- 面（カード/パネル）
- 文字（通常/補助）
- 強調（主要操作、警告、エラー）

```css
:root {
  /* surfaces */
  --ui-bg: #f6f7f9;
  --ui-surface: #ffffff;
  --ui-surface-muted: #f1f5f9;

  /* text */
  --ui-text: #0f172a;
  --ui-text-muted: #475569;

  /* borders (UI境界として使うものは 3:1 を下回らない濃さにする) */
  --ui-border: #838da0;
  --ui-border-strong: #64748b;

  /* primary */
  --ui-primary: #1d4ed8;
  --ui-primary-contrast: #ffffff;

  /* focus */
  --ui-focus-ring: rgba(37, 99, 235, 0.55);
  --ui-focus-shadow: 0 0 0 3px rgba(59, 130, 246, 0.35);
}
```

補足: 上記 `--ui-border` は白背景に対して約 `3.34:1`、`--ui-bg` に対して約 `3.12:1` を目安とする。

### 文字

- 本文サイズは `14-16px` を基本にする
- 行間は `1.5-1.7` を目安にする
- 見出し階層はサイズ差より、太さと余白で作る
- 薄い文字色（`#94a3b8` 相当）は本文に使わず注釈限定にする
- 数字列（表/ID等）は必要に応じて等幅数字を使う

### 余白

- 余白単位は `4px` 刻みに寄せる
- まとまりは内側余白で作り、境界線の増加で解決しない
- 同種コンポーネント間の余白は統一する
- 例外値を使う場合は理由をコメントする

### 角丸

角丸スケールは以下に固定する。

- 入力欄、ボタン、タブ: `8px`
- カード、パネル: `12px`
- モーダル: `16px`
- タグ、件数バッジ: `999px`（ピル形はこの用途のみ）

### 影

影は3段階のみ使用する。

- 影なし（標準）
- 薄い影（カードを浮かせる場合のみ）
- 強い影（モーダル等の前面要素のみ）

### 状態（ホバー、フォーカス、押下、無効）

- 通常、ホバー、押下、無効、フォーカスの見た目を必ず用意する
- キーボード操作の `:focus-visible` を必ず可視化する
- 状態変化を色だけで表現しない（下線、枠線、アイコン、太字を併用）
- 動きは短く控えめにする（`120-200ms`）

## コンポーネント規約

### ボタン

- 種類は原則3種類に限定する
- 主要ボタン: 塗り（`--ui-primary`）
- 通常ボタン: 枠線（`--ui-border`）
- 補助ボタン: 文字（背景なし）
- 角丸 `8px`、最小高さ `36-40px`
- クリック対象 `24 x 24` CSS px 以上

### 入力欄（テキスト、セレクト等）

- 枠線コントラストは `3:1` 以上を維持する
- エラー表示は枠線色だけで終わらせず、説明文を併記する

### ナビゲーション（トップバー、タブ）

- 選択中は背景色だけでなく、下線や太字を併用する
- タブやナビをピル形にしない
- 背景グラデーションは原則使わない
- クリック対象を `24 x 24` CSS px 未満にしない

### バッジ、タグ、件数表示

- ピル形はタグ用途に限定する
- バッジは短い文言のみ
- 状態差分は色だけでなく文言も併用する
- バッジ文字のコントラストは `4.5:1` 以上を確保する

### 一覧（患者一覧など）

- 行の高さを揃え、区切り線の濃度を安定させる
- 選択行は背景色と左強調線などを併用する
- 操作列はボタン規約を適用する
- 重要列は太さと余白で優先順位を作る

### モーダル、通知

- モーダル: 角丸 `16px`、強い影のみを使用
- 通知色は意味（成功、警告、エラー）と一致させる
- 通知は色だけでなく見出し文言やアイコンを併用する

## 運用ルール（崩れない仕組み）

- 新しい色や余白を足す前に、既存トークンで表現できるか確認する
- 画面固有CSSで新しい角丸や影を増やさない
- 必要ならトークンを追加し、用途を明示する
- Storybook に主要部品（ボタン、入力、タグ、通知、カード）を揃え、見え方を常時確認できる状態を維持する
- 変更レビュー時は「コントラスト比」「クリック対象サイズ」をチェック項目へ必ず含める

## 適用開始時の実務チェック

- 新規UI・改修UIで本ガイドラインへの適合確認を実施したか
- 直接値（色、余白、角丸、影）を書かず、トークン参照を優先したか
- フォーカス可視化と状態差分を色以外でも判別できるか
- 主要操作と補助情報の視覚優先順位が分離されているか
