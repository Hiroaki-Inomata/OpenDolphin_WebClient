# Codex Automation Prompts Revised

## 目的
この文書は、`docs/server-modernization/planning/codex_automation_workplan_revised.md` の各タスクを実行するときに参照する補助指示集である。  
進捗判定の第一正本は `codex_automation_workplan_revised.md` であり、この文書はそれを補助する。

## 使い方
- まず `codex_automation_workplan_revised.md` で未完了先頭タスクを特定する
- ここでは、そのタスク ID に対応する節だけを参照する
- 指示が広すぎる場合は、2 時間以内に終わる範囲へ自分でさらに狭めてよい
- blocker 条件に当たる場合は、その時点で停止する

## 共通ルール
- `target/` は編集しない
- `tools/flyway/sql` を migration 正本とする
- `src/main/resources/db/migration` は必要に応じて同期する
- source 不足、仕様未確定、変更範囲外テスト失敗時は停止する
- 後方互換性は考慮しない
- ただし、現時点で切ると仕様判断が必要な契約は、根拠なしに削らない

## P2-01 ORCA transport の設定キャッシュと client 寿命分離
### 目的

- `server-modernized/src/main/java/open/dolphin/orca/transport/OrcaHttpClient.java`

### やること
- どこで設定を読み直しているか確認する
- どこで `HttpClient` / `OrcaHttpClient` を作っているか確認する

### 完了条件
