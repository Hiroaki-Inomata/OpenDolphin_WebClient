# server-modernized 一時据え置き領域（P0-06）

- 更新日: 2026-03-11
- RUN_ID: 20260310T203518Z
- 目的: 刷新の主経路を止めないため、今回スコープ外（据え置き）と後続着手候補を明確化する。
- 前提: `docs/modernization/architecture-principles.md` の固定方針に従う。

## 1. 一時据え置き（今回スコープ外）
| 区分 | 対象 | 据え置き理由 | 再開トリガー |
|---|---|---|---|
| 帳票出力 | `reporting/**` / `open/dolphin/reporting` 相当 | 主要業務（受付・カルテ・ORCA・添付）の刷新を優先し、帳票刷新は依存先確定後に着手するため | P3（module境界）とP6（データモデル）が安定した時点 |
| 低影響ユーティリティ | 直接業務に影響しない補助共通部品 | 先行着手しても主経路のリスク低減効果が薄く、並行改修でノイズになるため | 主経路の退行監視（P1-10）確立後 |
| UI上の体裁調整のみの改修 | server-modernized の業務契約に影響しない表示差分 | サーバー刷新タスクと独立に計画できるため | API契約固定（P2/P3）後に別トラック化 |

## 2. 後続着手候補（据え置き解除順）
1. 帳票出力経路の契約明確化（入力DTO/出力形式/監査要件の固定）
2. `reporting/**` の依存棚卸し（entity直参照、ローカルファイル依存、旧 converter 依存）
3. 帳票生成の保存/配布先を新方針へ統一（DB/オブジェクトストレージ/集中ログ）
4. 低影響共通部品の整理（重複・未使用・旧互換残骸）

## 3. 着手禁止ルール（現時点）
1. P1〜P3 の主経路タスクを阻害する修正要求がない限り、据え置き対象へ着手しない。
2. 据え置き対象で障害対応が必要な場合は、修正を最小化しWBSへ例外記録を残す。
3. 据え置き解除は、WBS上で依存完了を確認してから行う。

## 4. 影響管理
- 据え置き対象に起因する既知リスクは、障害管理票とWBSの該当行へ紐づける。
- 仕様問い合わせが入った場合は「据え置き対象」であることを即時回答し、主経路タスクへの横入りを防ぐ。
